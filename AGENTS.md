# AGENTS.md

本文件是本项目的 AI 开发薄入口。详细规则、上下文路由、整体设计、模块设计、项目规划和开发规范沉淀在 `.harness/` 目录中。

## 必读顺序

1. 先读 `.harness/README.md`，理解 Harness 结构和工作流。
2. 再读 `.harness/routing/context-map.md`，根据任务关键词定位需要读取的上下文。
3. 涉及实现前，读取 `.harness/docs/` 下对应的整体设计、模块设计和开发规范。
4. 任务收尾前，读取 `.harness/changes/README.md` 判断是否需要记录变更。

## 项目定位

本项目计划实现一个数据库洞察 MCP 项目，面向后端 AI 开发过程。核心目标是弥补 AI 无法感知数据库层上下文的问题，让 AI 在开发时能了解表结构、索引设计、数据量估算、查询执行计划和是否可能走慢查询，从而辅助后端代码和 SQL 设计。

## 已确认方向

- 第一版支持 MySQL / MariaDB。
- 第一版使用 Python。
- 本地安装使用和服务器部署团队使用都要支持。
- 本地优先使用 MCP stdio。
- 服务器部署预留 HTTP/SSE 能力。
- 配置支持 CLI 主路径、配置文件和环境变量覆盖。
- 敏感配置需要加密能力。
- 安全策略为“推荐只读账号 + 工具只暴露白名单能力”，即使账号权限较大，工具也不提供高风险操作入口。
- EXPLAIN 只允许 SELECT。
- 第一版工具范围为开发辅助版：表结构、索引、行数/容量估算、SELECT EXPLAIN、相关表识别和基础风险提示。


## 维护原则

- `AGENTS.md` 只维护入口、读取顺序。
- 长期规则不要堆在本文件中，应沉淀到 `.harness/`。
- 开发规范、领域模型、模块边界、注释和可读性要求以 `.harness/docs/standards/` 为准。
- 当项目规划、产品方案、模块设计、安全规范或部署策略变化时，同步更新 `.harness/docs/` 中对应文档。
