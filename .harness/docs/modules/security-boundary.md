# Security Boundary Domain Model

## 核心原则

本项目不是通用数据库客户端，而是面向后端 AI 开发过程的数据库洞察工具。安全边界必须默认收紧：只暴露项目需要的固定能力，完全杜绝其他数据库操作入口。

## 角色

- AI Client：调用 MCP 工具的 AI 开发工具。
- MCP Server：接收请求并执行安全校验。
- Database Account：用户配置的数据库账号，推荐为只读账号，但工具安全不依赖账号权限是否足够小。
- Safety Guard：SQL 解析、能力白名单判断和危险操作拦截逻辑。

## 允许行为

- 读取 MySQL / MariaDB 元数据。
- 读取表结构、字段、主键、索引、行数估算和容量估算。
- 由工具内部对单条 SELECT SQL 执行 EXPLAIN。
- 对带占位符但未提供参数的 SELECT 做结构分析，并跳过 EXPLAIN。
- 返回基础风险提示。

## 禁止行为

- 执行 INSERT、UPDATE、DELETE、DDL、CALL、LOAD DATA 等语句。
- 执行多语句 SQL。
- 接受用户传入的 EXPLAIN 语句。
- 为了执行 EXPLAIN 自动猜测或编造参数值。
- 提供通用 SQL 执行入口。
- 返回明文数据库密码、master key、完整连接串。
- 在未确认设计前引入自动改表、自动建索引、自动修复 SQL 等能力。

## 防护层次

1. MCP 工具层只定义固定工具，不提供任意 SQL 执行工具。
2. 数据库访问层只暴露白名单查询能力。
3. SQL 解析层只允许单条 SELECT；EXPLAIN 由工具内部生成。
4. 文档推荐使用只读数据库账号，但不把账号权限检查作为一期核心能力。
5. 输出层脱敏敏感信息。
